My DITA learnings

Introduction to using DITA for documentation projects and the benefits of a documentation culture.

Key points

This page captures some of my DITA learnings in the hopes of helping others use DITA.

DITA is a framework that helps writers contribute to a specific writing project even if the writers are spread across different geographies, have different writing styles and are unfamiliar in using appropriate information types, among other things.

By using DITA, you can enforce a standard topic layout, elements that conform to that topic and so on. For example, writers who need to explain a concept, can choose to use a conceptual dita topic that allows them to introduce only writing elements suitable for that topic. You can customize the DITA templates to provide such restrictions.

In today's world of open source development tools and infrastructure, many organizations are moving away from using DITA for their inhouse documentation needs. Product companies are trying to foster a documentation culture within their development teams. This can help companies bridge knowledge gaps between teams, eliminate information silos, and increase new engineers' productivity.

To motivate engineers create documentation as part of their development process, teams support tools such as markdown or a wiki implementation. However, implementing Wiki comes with its own set of drawbacks.

DITA best practices

To get the most out of DITA, consider following these industry best practices:

  • Topic-Based Authoring: Ensure you are using the correct information types (Concept, Task, Reference). Concepts explain "what," Tasks explain "how," and References provide lookup data.
  • Content Re-use (DRY): Use conref to share content between topics. This prevents duplication and ensures that updates to common notes or warnings only need to be made in one place.
  • Semantic Inline Elements: Instead of generic styling, use semantic tags like <uicontrol> for UI elements, <filepath> for file paths, and <userinput> for terminal commands.
  • Metadata and Prolog: Use the <prolog> section to add keywords and metadata. This significantly improves the accuracy of the built-in search engines in WebHelp and other outputs.
  • Relationship Tables (RelTables): Manage links in the DITA Map using relationship tables rather than hardcoding cross-references. This makes your content more modular and prevents broken links if topics are moved or removed.
